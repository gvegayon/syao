Example 1

Adapted from From Ch. 3 of BH

An airline overbooks a flight, selling more tickets for the flight than there are seats on the plane (figuring that its likely that some people wont show up). The plane has 105 seats, and 110 people have booked the flight. Each person will show up for the flight with probability 0.95, independently. Find the probability that there will be enough seats for everyone who shows up for the flight.

Example 1: Solution using math

  • What is this distribution?

  • Using R, how can we solve this?

  • Since \(X\), the number of passangers that show up, is binomial with parameters (110, .95), the probability that the flight is room for all is the sum of pmfs

    n <- 110
k <- 105
p <- .95
idx <- (k + 1):n 
1 - sum(choose(n, idx)*p^idx*(1 - p)^(n - idx))

  • How about using a single line of code?

    pbinom(105, 110, .95)

Example 1: Solution using simulations in R

  • This is more interesting!

  • We need to make enough experiments so we can approximate hte solution

  • Here is some pseudo-code you can follow:

    1. Set up the seed and the parameters.

    2. Write out the function that retunrs the statistic of interest

    3. Put that function in a for-loop / or similar, and repeat the experiment thousands of times.

    4. Return and avarage out the results.

Example 1: Solution using simulations in R (cont d)

# 0. Seed and parametrs
n <- 110
k <- 105
p <- .95

# 1. Simulation that performs the experiment
sim <- function(n) {
  arrive <- sample(0:1, n, TRUE, c(1-p,p))
  sum(arrive)
}

# 2. Do the for-loop
res <- replicate(50000, sim(n))

# 3. Return and average the results
mean(res <= k)  # Obtained
## [1] 0.64774
pbinom(k, n, p) # Expected
## [1] 0.6484899

Example 3: Monopoly

Mono-what?

In the game, players roll two six-sided dice to move around the game board, buying and trading properties, and developing them with houses and hotels. Players collect rent from their opponents, with the goal being to drive them into bankruptcy. (wiki)

Example 3: Rules of Monopoly

What should be the best strategy to play Monopoly (in terms of property purchase)? A question that has keep be awake several nights. The idea is to create a simulation function that provides a solution to this problem. To do so, you need to consider the following:

  1. Users throw 2 six-sided-die at the same time to advance.

  2. There are 40 possible spots to fall on.

  3. When users fall in chance/community chest spots, there are some cards that may ask them to move around the board (that changes everything!).

  4. If the player hits double three times in a row, he goes imidiately to jail (more on die rules here).

To ease the simulation, assume that:

  1. Theres only a single player (saad)

  2. Tthe player has infinite resources (in other words, dont mess with bankrupcy rules), and

  3. The game ends at the turn number 500.
